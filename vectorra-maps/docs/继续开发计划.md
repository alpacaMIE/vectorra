# Vectorra SDK Android 1.0 继续开发规划（Phase-Task）

## Summary

本文件为路线摘要。正式执行计划文件：`docs/superpowers/plans/2026-06-03-vectorra-sdk-android-1-0-continuation-roadmap-plan.md`。

规划采用 `Phase -> Task -> 验收` 结构，按 Android 1.0 路线推进：先前置状态语义、native 事件通道、`TileCacheStore` 内部骨架，并固化 GLB/GLTF、raster、DEM、MBTiles 基线；再实现 3D Tiles Runtime、MVT 渲染与查询、离线缓存产品化；最后做 1.0 发布硬化。

## Phase-Task Backlog

### Phase 0 - 渲染链路固化

| Task | 内容 | 验收 |
| --- | --- | --- |
| P0.T0 状态语义与事件通道 | 定义 layer/source 状态 payload、resource kind、source/layer id、generation、错误、来源、线程投递，以及 loading/loaded/failed/removed 对 raster、DEM、model、MBTiles 的语义 | P0.T1/T2 不再需要补状态身份、生命周期或 native callback 设计 |

### Phase 1 - 3D Tiles Runtime Beta

| Task | 内容 | 验收 |

| P1.T1 API 定义 | 新增正式 `Vectorra3DTilesSource`、`Vectorra3DTilesLayer`、`Vectorra3DTilesOptions`；不扩展 `add3DTilesModelLayer` | source/layer/options API 可编译；sample 使用正式路径；状态接入统一 layer/source 契约 |
| P1.T2 Tileset 加载 | 支持 URL/本地 `tileset.json`，复用 base URI 解析 content URI | 加载失败进入统一 source/layer error；本地/远程 URI fixture 一致 |
| P1.T3 Runtime Tile State 与 Traversal | 建立 3D Tiles runtime tile state；实现 camera-driven traversal、SSE/LOD、视锥/距离筛选、请求优先级、unload budget，并支持 ADD/REPLACE | root/child/ADD/REPLACE fixture 通过；pan/zoom 后 tile load/unload 正确；超过预算有明确状态 |
| P1.T4 GLB/GLTF Content 生命周期 | 加载 `.glb/.gltf` tile content，复用 headers、interceptor、cache，支持取消、去重、unload | 同一 tile content 不重复加载；unload 后不再渲染或保留查询状态 |
| P1.T5 Transform 与 Bounds | 应用 tileset transform、tile transform、`RTC_CENTER` 合成后的 renderer transform 与 visibility；使用 region/box/sphere 做 debug bounds 与基础筛选 | 经纬度、高程、姿态、transform chain、`RTC_CENTER`、WGS84/ECEF bounds fixture 通过 |
| P1.T6 b3dm 渲染支持 | 解析 b3dm header、feature table、batch table、内部 GLB payload；正确应用 `RTC_CENTER` | b3dm fixture 覆盖长度、padding、GLB offset、`BATCH_LENGTH`、`RTC_CENTER` 和渲染路径 |
| P1.T7 P1 设备 smoke | sample 通过 `Vectorra3DTiles*` 加载 building/model tileset，执行 native 生命周期验证 | UI 无明显卡顿；remove/re-add、旋转或 pause/resume、截图、销毁重建无崩溃；错误 tileset 可见 |
| P1.T8 P1 验收 | 汇总 API、fixture、设备验证结果 | 3D Tiles 正式入口只有 `Vectorra3DTiles*`；b3dm + `RTC_CENTER` 无 P0/P1 blocker |

## Public API Changes

- 新增公开 layer/source load/error 状态契约，最小形状为监听 API + 当前状态查询 API；状态值覆盖 loading/loaded/failed/removed，供 sample、业务方和测试共同使用。
- 新增正式 3D Tiles source/layer/options API；`add3DTilesModelLayer` 从 P0 起只保留为 deprecated/experimental smoke API，不承载 1.0 正式 3D Tiles runtime 能力。
- 新增 vector tile source 和简化 MVT style layer API，不实现完整 Mapbox Style JSON。
- 新增 offline manager、offline region、prefetch task、cache status API；offline API 不暴露 proxy/native 细节，cache status 由 `TileCacheStore` 支撑。
- 新增 internal `TileCacheStore` skeleton/adapter 作为 P1/P2 请求缓存 owner 的前置依赖；P3 再产品化 offline/cache 管理 API。

  - `.\gradlew.bat :vectorra-maps:testDebugUnitTest`
  - `.\gradlew.bat :vectorra-maps-turf:testDebugUnitTest`
  - `.\gradlew.bat :vectorra-sample:assembleDebug`

  - `.\gradlew.bat :vectorra-maps-turf:publishReleasePublicationToMavenLocal`

- P0、P1、P2、P3、P4 额外执行 native 设备 smoke 矩阵：
  - 记录 device model、Android API、ABI、GPU/Vulkan。
  - 覆盖 cold start、加载成功、坏 URL/坏 tileset 错误、visibility、remove/re-add、rotate 或 pause/resume、snapshot 非空、destroy/recreate。
  - P0 覆盖统一 layer/source 状态、native 状态 JNI 回调、坏 GLB/坏 URL UI。
  - P1 覆盖 b3dm 渲染和 `RTC_CENTER` 正确定位。
  - P2 覆盖 MVT pan/zoom、query、tile unload 后不再渲染或命中。
  - P3 覆盖离线 raster MBTiles、MVT MBTiles、cleanup 和 remove/re-add。
  - P4 覆盖 `arm64-v8a` 与 `x86_64` 打包产物，并至少一台真实 Vulkan Android 设备完整 smoke。

## Assumptions

- 平台范围限定为 Android 到 1.0；iOS/桌面不进入本规划。
- 时间口径为无日期里程碑，按 Phase 0 到 Phase 4 顺序推进。
- 本规划不要求修改 `vectorra-references/`；参考资料只用于阅读和对比。
- GLB/GLTF 视觉 model rendering 已修复，但必须在 P0 通过统一状态与设备回归固化。
- 3D Tiles 首版优先 building/model tileset；P1 必须支持 b3dm 渲染并正确应用 `RTC_CENTER`，不做 pnts、i3dm、cmpt、implicit tiling、metadata styling。
- b3dm 内嵌 GLB payload 在 P1 固定落 cache URI 后复用 renderer，不新增 native bytes 渲染通道。
- MVT 首版只做简化渲染与查询，不做完整 Mapbox Style JSON。
